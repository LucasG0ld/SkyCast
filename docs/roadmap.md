# Development Roadmap - SkyCast

## Sprint 1: Foundation & Data (Week 1)
- [ ] Initialize Expo TypeScript project.
- [ ] Set up Zustand store (initial state: cities, units, language).
- [ ] Create API service for WeatherAPI.
- [ ] Implement GPS location fetching.

## Sprint 2: Navigation & Search (Week 1-2)
- [ ] Setup React Navigation (Tabs + Stack).
- [ ] Build the "Search Page" with auto-complete.
- [ ] Implement "Swipe" logic between multiple city cards.
- [ ] Create the "Empty State" screen (Welcome screen).

## Sprint 3: Immersive UI & Backgrounds (Week 2-3)
- [ ] Create the Dynamic Background engine (Weather code -> Image mapping).
- [ ] Design Glassmorphism cards for weather details.
- [ ] Integrate Lottie animations for weather icons.
- [ ] Implement the progressive "Startup Loader".

## Sprint 4: Settings & Offline Logic (Week 3)
- [ ] Implement i18n (FR/EN) translation system.
- [ ] Add C°/F° unit conversion logic throughout the app.
- [ ] Set up Theme Switcher (Auto/Light/Dark).
- [ ] Add Offline Mode banner and AsyncStorage caching.

## Sprint 5: Landing Page & Deployment (Week 4)
- [ ] Develop the React Landing Page.
- [ ] Generate Android APK via Expo EAS.
- [ ] Host the Landing Page and upload the APK to GitHub Releases.
